- ✅ **2026-05-25 / 2026-05-26 (CTA contrast root-cause incident — 6-demo + scaffolds + rules sweep)** — User reported "dark text on dark bg" on multiple primary CTAs across the 3 new demos (lawyer / yoga / barber). Two-layer bug, only the surface layer was caught initially:
  - **Surface layer (fixed first):** `bg-accent text-bg hover:bg-accent-deep` (darken-on-hover without text-color update) produced muddy hover states + the yoga primary Button failed WCAG AA in DEFAULT at 3.0:1 (terracotta + lilac). ~25 buttons replaced via sed across 6 demos with the inverted pattern `bg-accent text-bg hover:bg-text hover:text-bg`.
  - **Root layer (caught via browser-inspection):** EVERY primary CTA across all 6 demos still rendered wrong colors. Lawyer "Leistungen ansehen" computed `color: rgb(11, 42, 31)` (forest) on a forest-green background — INVISIBLE. Cause: `body { color: var(--color-text) }` was declared UNLAYERED in every demo's `src/styles/global.css`. Per CSS @layer cascade rules, unlayered rules win over `@layer utilities` regardless of specificity → body's inherited color silently shadowed every `.text-bg` / `.text-X` utility.
