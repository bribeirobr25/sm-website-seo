# education.md — Education & Kids Vertical Template
## Daycare · Preschool · Tutoring · Language School · Music Lessons · After-School Programs

**Applies to:** Education clients across product types 1–3 (per `TECH.md` §1). Single-instructor music/language lessons map to `templates/studio.md` Solo Instructor archetype; daycare/tutoring centers are closer to `templates/health.md` Solo Practitioner trust patterns + parent-trust messaging. Multi-location language-school chains are Type 3 (booking-DB + multi-locale + course catalog).

**Reference research:** Based on `local_business_website_benchmark_report.md` §7.11 (Wizard, CNA, Cultura Inglesa, Maple Bear, Kumon, Mathnasium, Goddard, KinderCare, Sylvan, Bright Horizons, EF, Berlitz, British Council, Stagecoach).

**Use this doc as a moodboard with principles, not a layout to clone.** If two clients in the same vertical end up with the same hero pattern, the template has failed at its job.

---

## Rules at a glance

- **The buyer is the parent, not the student.** Even for adult language students, the cognitive register on landing is "decision time, can I trust this place with hours of my life or my child's?" Trust precedes everything.
- **Pick one of four archetypes** before designing: **Premium Daycare/Preschool Network**, **Tutoring Franchise**, **Language School / Adult Education**, or **Solo Tutor / Music Teacher / Independent Instructor**. ~70 % of agency clients fall into Solo Instructor.
- **Trial / placement / consultation is the conversion path.** Education buyers rarely commit on first visit — they want a sample. Surface the trial-class / placement-test / consultation CTA above the fold.
- **Programs organized by age/level**, not by discipline. "Beginner (age 6-8)" beats "English course." Parents/students search by age + level first.
- **Safety + accreditation is mandatory** for kids programs. Visible accreditation badges (NAEYC in US, Ofsted in UK, MEC-credenciado in BR), background-check policy, ratios (1 teacher : 6 toddlers), emergency procedures.
- **Teacher/instructor profiles matter.** Education is high-trust + person-to-person. Show real instructors with real credentials.
- **Tone must be warm and competent.** Cold/clinical reads "drill-and-kill." Cutesy reads "amateur." Aim middle.
- **Avoid the smiling-multi-ethnic-children-stock-photo.** It's the loudest tell in this vertical.

### Solo-Operator meta-archetype (cross-vertical pattern)

Most agency Education clients (solo tutor, single-instructor music teacher, individual language coach) are the Education-specific implementation of the **Solo-Operator meta-archetype** in `DESIGN-BEST-PRACTICES.md` §3:

- Operator name in headline ("Ana Costa — Professora de Piano em Niterói")
- Portrait of the operator with student (consent-cleared) or with instrument/materials
- ONE primary CTA (book trial class / request consultation / WhatsApp)
- Credentials signature line (degree + years teaching + age groups + certifications)
- Program list (5–10 programs by age/level, not a card grid)
- Trial-class explainer (free/paid + duration + what to expect)
- Service area + delivery format (in-person / online / hybrid)
- Pricing reassurance ("Primeira aula grátis" / "First class free")

### Sourcing rules (apply before any visual decision)

- **Photo + favicon sourcing:** `DESIGN-BEST-PRACTICES.md` §3 — 8-tier photo + 5-tier favicon priority. **Education-specific:** classroom/student photos are extremely high-value but require **explicit written consent** for minors (parental consent + photo release). Booking platforms (Superprof / Preply / italki for tutors; Sympla/Eventbrite-style for short courses) are tier 3 alternatives. Instagram is tier 4.
- **Color palette sourcing:** `DESIGN-BEST-PRACTICES.md` §5 — 6-tier color source hierarchy.
- **Prospect intake template:** `CHECKLIST.md` §9.

## 1. The four Education archetypes

| Archetype | Brand priority | Primary CTA | Production cost |
|-----------|---------------|-------------|-----------------|
| **A. Premium Daycare/Preschool Network** (KinderCare, Goddard, Bright Horizons, Maple Bear) | Parent-trust, accreditation, safety, programs by age | Schedule a tour | High — center-finder + tour scheduling + parent portal |

## 2. Information architecture per archetype

### Archetype A — Premium Daycare/Preschool Network

1. **Hero** — photo of children engaged in real activity (not staged group photo) + headline addressing parent anxiety ("Where your child is known by name") + ONE CTA ("Schedule a tour").
2. **Programs by age** — Infant / Toddler / Preschool / Kindergarten / After-school, each as a card with age range + ratio + brief description.
3. **Curriculum/method** — Reggio / Montessori / play-based / academic — what method, why, evidence.
4. **Safety + accreditation** — accreditation badges, ratios, background-check policy, emergency procedures, parent communication tools.
5. **Center locator** — map + filter by services/ages.
6. **Tour booking** — date picker + center selection.
7. **Parent stories / testimonials** — verbatim quotes + photos (consent-cleared).
8. **Footer** — heavy: accreditation links, regulatory compliance per state/country, careers, accessibility.

### Archetype B — Tutoring Franchise / Center-Based Learning

1. **Hero** — photo of student-tutor interaction (consent-cleared) + headline addressing parent goal ("Help your child master math, one step at a time") + ONE CTA ("Schedule an assessment").
2. **Method explainer** — Kumon's "self-learning" / Mathnasium's "method" — proprietary differentiation.
3. **Programs by age + subject** — Math / Reading / Writing / Test Prep + age ranges.
4. **Results / outcomes** — percentile improvements, grade-level advancements, college-acceptance stats.
5. **Center locator** — local centers + assessment booking.
6. **Assessment process** — explain what happens at the assessment, removes friction.
7. **Pricing transparency** — typical monthly cost or starting-from price.
8. **Parent testimonials + before/after stories**.
9. **Footer** — franchise/corporate chrome.

### Archetype C — Language School / Adult Education

1. **Hero** — image of students in conversation (real, consent-cleared) + headline about language journey + ONE CTA ("Find your level — free placement test").
2. **Course catalog** — by language × level × format (in-person / online / hybrid) × duration.
3. **Method differentiator** — communicative / immersive / structured — pick one + explain.
4. **Locations / schedule** — campus locator + class times.
5. **Placement test** — free, online, builds personalized recommendations.
6. **Pricing tiers** — per-course or membership-style.
7. **Student outcomes** — Cambridge / IELTS / TOEFL results, real student testimonials.
8. **Footer** — accreditation (Cambridge English, Goethe, Cervantes for IBR/EU markets) + careers + multi-language site switcher.

### Archetype D — Solo Tutor / Music Teacher / Independent Instructor (the agency's default)

1. **Top bar** — phone + WhatsApp + (kids tutor) parent line.
2. **Hero** — operator portrait + instrument/materials/student (consent-cleared) + headline naming operator + discipline + age range + city ("Ana Costa — Aulas de Piano para crianças e adolescentes · Niterói"). ONE CTA ("Aula experimental grátis").
3. **Credentials signature line** — degree + conservatory/university + years teaching + age groups + spoken languages + (for music) instruments.
4. **Programs by age/level** — typography list (not card grid). "Iniciantes (6-10 anos) · Intermediário · Preparação para conservatório · Adultos."
5. **About / Bio** — single paragraph + portrait. "Why this teacher, why this method."
6. **Method / approach** — 2-3 sentences. What's distinctive.
7. **Trial-class explainer** — duration + free/paid + what to expect.
8. **Reviews** — parent + student testimonials, 3–4 verbatim quotes.
9. **Schedule / availability** — days + times + format (in-person at student's home / instructor's studio / online).
10. **Footer** — Impressum/Política/legal + credentials registration + child-protection statement if applicable.

## 4. Photography direction

### Universal Education rules

- **Real students with consent.** Photos of minors require parental + photo-release consent in writing.
- **Real activity, not staged.** Children actually painting / building / reading / playing music. Not the smiling-around-a-table-with-folders stock photo.
- **Operator + student.** Like Pets (vet+pet) and Beauty (stylist+client), the relationship is the product.
- **Real classroom / studio / center.** Not stock interior shots.
- **Avoid the "diverse children stock kit"** — same diverse-children-collage across thousands of education sites.

### Per-archetype photography notes

| Archetype | Photo style |
|---|---|
| A — Daycare/preschool | Real children in unscripted activity · indoor + outdoor play · meals · rest · teachers engaged with children |
| B — Tutoring franchise | Tutor + student in genuine learning moment · real center interior · before/after work samples |
| C — Language school | Students in conversation · campus interiors · cultural events · authentic learning moments across age groups |
| D — Solo instructor | One operator portrait · one teaching-in-action shot (consent-cleared) · workspace/instrument detail |

## 5. Typography pairings

### What works in Education

| Pairing | Display | Body | Mood |
|---|---|---|---|
| **Premium daycare warmth** | Fraunces · Source Serif Pro | Inter | KinderCare/Goddard register — warm + trustworthy |
| **Tutoring franchise** | Inter Display / Söhne | Inter | Kumon/Mathnasium — method-driven, results-focused |
| **Language school heritage** | Tiempos · Lyon · Cormorant | Inter | Berlitz/British Council — institutional + multilingual-friendly |
| **Solo instructor warmth** | Fraunces · Cormorant | Manrope | Standard solo — approachable + competent |

### Rules

- **Never Comic Sans, never Curlz, never Chalkboard.** Cute-education typography is the visual equivalent of treating customers like children.
- **Multilingual-friendly characters required** for language schools — ensure diacritics (á, é, í, ó, ç, ñ, ü, ß) render properly.
- **Cap to two families.**

## 7. Copy voice cues

### What to say

- **Name the operator + discipline + age range + city** in the opening. "Ana Costa — Aulas de Piano para crianças e adolescentes · Niterói."
- **Quantify experience.** "10 anos formando músicos" / "200+ alunos preparados para conservatório."
- **Lead with trial-class reassurance.** "Primeira aula grátis, sem compromisso."
- **Specify age ranges per program.** "Iniciantes (6-10 anos)" beats "Aulas para crianças."
- **Cite results specifically** for tutoring/test-prep. "92 % dos alunos avançaram um nível em 6 meses."
- **Method differentiation in one sentence.** "Método baseado em [approach] com foco em [outcome]."

### What never to say

- ❌ "Where every child shines." Universal cliché.
- ❌ "Quality education guaranteed." Empty.
- ❌ "Building tomorrow's leaders." Performative.
- ❌ "We believe every child has potential." Of course you do — everyone says it.
- ❌ "Award-winning [program]" without naming the specific award.
- ❌ Cutesy alliteration ("Math Magic," "Wonderful Wizards of Words"). Cringe.

## 8. Education-specific anti-patterns

| Anti-pattern | Why it fails | Better |
|---|---|---|
| **Stock photo of diverse smiling children around a table** | Universally fake; the kit appears on 10,000 education sites | Real children, real activity, consent-cleared. Or Placeholder. |
| **Crayon-box primary-colors-everywhere palette** | Reads cutesy-amateur | Restrained palette with one warm accent |
| **Cute-typography logo (Comic Sans, Chalkboard, Curlz)** | Cringe; signals "we don't take ourselves seriously" | Restrained serif or clean sans + warm copy |
| **"Schedule a tour" CTA buried in nav** | Trial/tour is the primary conversion; bury it and lose the buyer | Hero CTA + sticky mobile CTA |
| **No accreditation badges visible** for daycare | Accreditation is the trust signal | Badge strip in trust section + footer |
| **No ratios visible** for daycare/preschool | 1:6 ratio is a parent's top question | Surface ratios per age group |
| **No safety/emergency policy** | Parents ask before booking; site doesn't answer = lose the buyer | FAQ + dedicated safety page |
| **Endless three-card grids without age filter** | Programs are age-specific; ignoring that wastes the buyer's time | Age filter first, then programs |
| **"Comprehensive curriculum" without specifics** | Empty | Name the method, the books, the framework |
| **Generic "Excellence in education" tagline** | Universal | City-specific or operator-specific tagline |

## 11. Measurement — KPIs that matter for Education

**Applies to:** every retainer-tier education client at production cutover. KPI framework, naming convention, and per-tier stack selection live in `KPI.md`; this section picks the 3–5 KPIs that matter most for tutors, schools, language centers, and instructors and how they wire.

**Legal callout:** Education clients with under-13 audiences trigger COPPA (US) and additional consent rules under DSGVO Art. 8 + LGPD Art. 14. See `LEGAL.md` §COPPA — under-13 audience. KPI events MUST NOT capture child identifiers; only aggregate counts.

### 11.1 Product KPIs

| # | KPI | Bucket | Source | Target / benchmark |
|---|-----|--------|--------|---------------------|
| 1 | Trial-class signup rate | Conversion | GA4 `contact_form_completed` (trial form) OR platform handoff | ≥ 6% of sessions |
| 2 | Trial → enrollment conversion (signed-up trial → paid enrollment) | Conversion (business) | CRM / platform / manual | ≥ 40% trial-to-enroll |
| 3 | Cohort retention (% students active month-over-month) | Retention | PostHog cohort (Tier 3) / platform data | ≥ 80% MoM for ongoing programs |
| 4 | Parent-vs-student source split (when audience is mixed) | Acquisition | GA4 source + device (mobile/desktop heuristic) | Vertical-dependent |
| 5 | Method-page engagement (% sessions reading the pedagogy page deeply) | Conversion (trust signal) | Clarity scroll depth on `/methode` or `/method` | ≥ 50% reach 75% scroll |

### 11.2 Per-tier stack

| Tier | Tools active | What it measures |
|---|---|---|
| Tier 1 + form endpoint (solo tutor with trial form) | GSC + Clarity + GA4 | KPIs #1, #4, #5 |
| Tier 2 (Astro — most common) | GSC + Clarity + GA4 | KPIs #1, #4, #5 + CRM for #2 |
| Tier 3 (school with LMS / Superprof / Preply integration) | GSC + Clarity + GA4 + PostHog + Sentry | All 5 KPIs |

### 11.3 Dashboard tiles

**GA4:** conversions by event (`trial_signup_completed`) · top landing pages by age/level (`/kinder`, `/erwachsene`, `/business`) · device split (parent-on-mobile vs student-on-desktop heuristic) · source/medium for trial signups.

**Clarity:** heatmaps on trial-signup CTA + method page + credentials page · scroll depth on long-form pedagogy explainer · recordings filtered to trial-form abandonment.

**PostHog (Tier 3):** trial → enrollment funnel · monthly retention cohorts · program-popularity ranking · drop-off-by-step analysis.

### 11.4 Vertical-specific event names

| Event | Fires when | Required params |
|---|---|---|
| `trial_signup_started` | Trial-class form first field focused | `program_slug`, `source_page` |
| `trial_signup_completed` | Trial form submitted (200 from endpoint) | `program_slug`, `student_age_band` (`under_13`, `13_18`, `adult`) — NO actual age, age band only |
| `method_viewed` | Pedagogy/method page LCP fires | `source_page` |
| `program_viewed` | Specific program/course page LCP fires | `program_slug`, `level` |
| `credentials_viewed` | Credentials/teacher-profile section enters viewport | `staff_slug` |

**COPPA / minors rule:** never capture child name, email, school, exact age, or any identifying parameter. Age bands only. Parent contact info stays in the form payload (server-side), never in the event payload.

| **Sentry** | Tier 2+ (full SDK) | Standard setup; `send_default_pii: false` critical when under-13 audience triggers COPPA |
| **PostHog** | Tier 3+ (school with LMS / Superprof / Preply integration) only | Trial → enrollment funnel; age-band-only parameters; no student-identifying data |
| **Neon** | Tier 3+ schools with own LMS | Students table (encrypted PII); attendance records. COPPA-relevant if under-13. |
| **Upstash** | Tier 2+ trial signup | Rate-limit 5/60s |
| **Booking platform** (Superprof / Preply / Tutor.com) | Optional — solo tutors | Deep-link with UTM-preserve handoff. Platform owns booking/payment infrastructure. |
| **Stripe** | Type 4+ — tuition subscriptions, lesson packs | SEPA + Pix per jurisdiction; SCA-compliant flow for EU |

**Minors rule:** every event captures `student_age_band` (`under_13`, `13_18`, `adult`) — never exact age. Never student name. Parent contact info stays in form payload (server-side); never duplicated to analytics events.

### 11.7 Share strategy

Per `SOCIAL-SHARING.md` §Per-vertical share strategy: **Medium leverage**.

- **Default targets:** WhatsApp + Copy-link + Facebook (parent-driven audience)
- **IG embed recommended:** ❌ No — under-13 audience caution; even consented student photos are a high-risk share target
- **Placement:** inline share at trial-CTA section (parents share with other parents) · footer fallback
- **OG image priority:** classroom-environment shot OR instructor portrait (with all-consent for any visible students). NOT diverse-children-stock. NOT identifiable students without explicit parental consent.
- **WhatsApp share copy:** "[School/tutor name] — [subject/program] in [city]. Free trial available." — invite framing

**Photo-consent rule:** any student photo on the site requires written parental consent on file. Document the consent process in the per-client `BRIEF.md`. Never use scraped or stock-with-faces children.

### 11.8 Schema.org variants

Use the most specific subtype:

- `EducationalOrganization` — generic
- `School` — primary / secondary school
- `Preschool` — daycare / kindergarten
- `MusicSchool` — music-focused
- `LanguageSchool` — language learning
- `CollegeOrUniversity` — higher ed (rare in agency scope)

```json
{
  "@context": "https://schema.org",
  "@type": "MusicSchool",
  "name": "[School name]",
  "address": { ... },
  "geo": { ... },
  "telephone": "+...",
  "openingHoursSpecification": [...],
  "hasCourse": [
    { "@type": "Course", "name": "Klavier", "courseCode": "PIANO-1", "audience": { "@type": "EducationalAudience", "audienceType": "Children" } },
    { "@type": "Course", "name": "Gesang", "courseCode": "VOICE-1" }
  ],
  "potentialAction": { "@type": "ContactAction", "target": "https://[trial-form URL]" }
}
```

`hasCourse` lists the offerings — long-tail ranking ("Klavierunterricht [stadtteil]"). Use `audience` to target search by age group.

### 11.9 GBP category + keyword pattern

- **GBP primary category:** `Tutoring service` / `Music school` / `Language school` / `Day care center` / `Preschool` (pick most specific)
- **GBP secondary categories:** subject + level (e.g., music school adds `Piano lessons`, `Voice lessons`)
- **Per-jurisdiction GBP attributes:** wheelchair-accessible, online appointments, online classes, free trial, accepts new students
- **Keyword pattern (DE):** `[subject] nachhilfe [stadtteil]` · `[instrument] unterricht [stadt]` · `sprachschule [stadt]`
- **Keyword pattern (BR):** `aula de [matéria] em [bairro]` · `professor de [matéria] [cidade]`
- **Keyword pattern (PT):** `aulas de [matéria] em [cidade]` · `explicações [matéria] [cidade]`
- **Example:** "Mathe Nachhilfe Mitte" · "Aula de violão em Copacabana" · "Explicações matemática Lisboa"
